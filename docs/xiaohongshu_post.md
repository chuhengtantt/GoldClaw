# 小红书文案：我用 Python 给 LLM 做了个「算盘」

---

## 标题候选

别再让 AI 算账了 —— 我用 Python 给 LLM 装了个「算盘」

---

## 正文

上周市场剧烈波动，我让 AI 帮我盯盘做交易决策。

结果发现一个致命问题：

**LLM 在复杂运算里会幻觉。**

简单的加减乘除还行，但一旦涉及：
- 1:20 杠杆的 CFD 盈亏计算
- 隔夜利息层层累积
- 止损止盈的阈值判断
- 连续多次交易后的资产复利

AI 就开始「一本正经地胡说八道」了。

一层算错，后面全错。而且你还看不出来，因为它输出的数字看起来都很合理。

---

### 为什么不用纯 AI ？

1. **幻觉问题**：数学公式层层堆叠之后，LLM 就算不准了，但输出结果看起来依然很自信
2. **Token 烧钱**：让 AI 每 3 分钟盯一次盘，每次都要把完整上下文喂一遍，一天下来 token 费用比交易手续费还高
3. **不可审计**：AI 脑子里的计算过程是黑盒，出了问题你根本不知道哪一步错了

---

### 我的方案：Python 算盘 + LLM 大脑

我做了一个叫 **GoldClaw** 的系统，核心思路很简单：

> **让 AI 只做决策，Python 负责所有计算和执行。**

```
金价 API → Python 状态机 → 计算盈亏 → 写入 SQLite
                                    ↓
                            更新信箱文件 → AI 读信箱 → 写决策
                                    ↓
                            Python 校验决策 → 执行交易
```

**分工明确：**
- Python：算账、盯盘、风控、数据存储 —— 准确、快速、免费
- AI：看数据、做判断、写决策 —— 发挥它的推理能力

---

### 最关键的：门铃通信系统

AI 和 Python 之间怎么通信？我用了一个「信箱 + 门铃」的设计：

**信箱（JSON 文件）：**
- Python 每 tick 更新 `state_for_openclaw.json`（当前金价、持仓状态、盈亏数据）
- AI 定时读取，写入 `orders_from_openclaw.json`（开仓/平仓决策）

**门铃（HTTP POST）：**
- 当金价出现异常波动（Python 检测到斜率突变），Python 会主动「按门铃」唤醒 AI
- AI 不需要一直盯着，被叫到再来看，大幅节省 token

**常规模式：** 15 分钟一次 tick
**异常模式：** 检测到波动 → 3 分钟一次 + 门铃通知 AI

这样 AI 不用 24 小时在线烧 token，只在关键时刻出手。

---

### 安全机制：Python 是唯一的「数据库管理员」

还有一个重要的设计：**AI 永远不能直接操作数据库。**

所有 AI 的决策都要经过 Python 的校验层：
- Pydantic Schema 校验（格式对不对）
- 权限校验（这个投资者能不能做这个操作）
- 幻觉检测（交易前后资产变化超过 50%？直接拒绝 + 记录到「耻辱柱」）

数据库是 SQLite，单文件，WAL 模式，Python 是唯一的写入者。AI 只能通过 JSON 文件「递纸条」，Python 校验通过才会执行。

---

### 结果

- AI 不再算错账了（Python 算的，精确到小数点后 2 位）
- Token 消耗降低 80%+（不需要每 3 分钟都调 AI）
- 所有交易有完整审计记录（SQLite + Dashboard 可视化）
- 极端行情时 Python 自动止损，不等 AI 来得及反应
